# 欢迎使用你的个人知识库

这个知识库采用了 **PARA 方法** 进行组织，旨在提高效率和清晰度。

## 📂 结构概览 (The PARA Method)

### 00-Inbox (收集箱)
- **用途**：快速捕捉。最初产生的所有内容（灵感、临时笔记、未分类文件）都放在这里。
- **原则**：保持清空。不要让文件在这里堆积，定期整理。

### 10-Projects (项目)
- **定义**：与目标相关联的一系列任务，有**明确的截止日期**。
- **示例**：“2024网站改版”、“日本旅行计划”、“撰写Q1季度报告”。
- **动作**：项目完成后，将其移动到 `40-Archives`。

### 20-Areas (领域)
- **定义**：你需要长期维护的责任领域，**没有截止日期**。
- **示例**：“健康管理”、“个人财务”、“职业技能”、“家庭生活”。
- **内容**：标准操作程序 (SOP)、长期积累的知识、体检记录等。

### 30-Resources (资源)
- **定义**：持续感兴趣的主题或题材，用于参考和学习。
- **示例**：“Markdown 技巧”、“食谱大全”、“读书笔记”、“设计素材”。
- **区别**：`Areas` 是你个人的责任（如“我的健康”），`Resources` 是外部的知识（如“健身知识”）。

### 40-Archives (归档)
- **用途**：存放已完成的项目或不再活跃的资源。
- **原则**：永远不要删除（除非是垃圾），而是归档。

### 90-Journal (日志)
- **用途**：每日日记、周记和定期反思。

### 99-Templates (模板)
- **用途**：存储系统核心模板。
- **注意**：已配置自动脚本，通常不需要手动修改这里。

## 💡 高效工作流方案

### 1. 每日循环 (Daily Loop)
1.  **开启日记**：点击“今天的日记”，自动生成当天的页面。
2.  **规划任务**：在日记的 `✅ 任务` 区域写下今日待办。
3.  **随手捕捉**：遇到任何新想法、新任务，按 `Ctrl+N` 新建笔记（自动存入 Inbox），快速记录后关闭，不打断当前工作。

### 2. 每周整理 (Weekly Review)
1.  **清空 Inbox**：
    *   是任务？ -> 移动到日记或具体的项目笔记中。
    *   是参考资料？ -> 移动到 `30-Resources`。
    *   是项目相关？ -> 移动到 `10-Projects` 对应文件夹。
    *   没用了？ -> 删除。
2.  **检查项目**：查看 `10-Projects` 里的活跃项目，更新状态。
3.  **归档**：将已完成的项目文件夹拖入 `40-Archives`。

### 3. 命名与链接规范
*   **文件命名**：简短且具有描述性。例如 `2024-01-21 会议记录` 或 `Python 基础语法`。
*   **双向链接**：
    *   提到某个概念时，用 `[[概念名称]]` 建立链接。
    *   在项目笔记中，链接相关的会议记录：`相关会议：[[2024-01-21 产品评审]]`。
*   **标签 (Tags)**：
    *   `#project/active` (进行中项目)
    *   `#todo` (待办)
    *   `#waiting` (等待中)

## 🛠️ 模板使用指南

系统已配置全自动化模板，以下是核心逻辑：

| 场景 | 操作 | 自动效果 |
| :--- | :--- | :--- |
| **写日记** | 点击左侧“日历/日记”图标 | 创建在 `90-Journal`，自动填入日期和任务栏 |
| **新项目** | 在 `10-Projects` 右键新建笔记 | 自动套用 `Project Note`，标题自动同步 |
| **开会** | 按 `Alt+E` 插入模板 -> `Meeting Note` | 自动填入日期、议程结构，标题自动同步 |
| **临时笔记** | 按 `Ctrl+N` (默认在 Inbox) | 自动套用 `Note` 基础模板 |

**关于标题的魔法**：
所有模板都使用了动态标题（`= this.file.name`）。这意味着你**修改文件名时，笔记内的标题会自动更新**，无需手动修改两遍。

## 🚀 进阶技巧

1.  **任务管理**：
    *   在任何地方输入 `- [ ] 任务内容` 即可创建任务。
    *   你可以使用 Tasks 插件（已安装）在日记里汇总所有未完成任务。
    *   输入 ````tasks` 然后回车，再输入 `not done` 再回车，最后 ```` 结束，即可查询所有待办。

2.  **Dataview 查询**：
    *   模板中已经预置了 Dataview 语法。随着你笔记增多，你可以用它来生成“所有进行中的项目列表”或“本周阅读的书籍”。
